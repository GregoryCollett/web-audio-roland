# Roland SH-2 Synthesizer Module — Design Spec

## Overview

Add a Roland SH-2-style synthesizer module to the existing workstation. The SH-2 features two oscillators, a noise mixer, an IR3109-style resonant lowpass filter with its own ADSR envelope, a separate VCA ADSR envelope, and a full LFO with multiple destinations. It shares the same monophonic 16-step sequencer pattern as the TB-303.

This spec also includes a component restructure — organizing all components into domain folders with shared primitives extracted.

## Architecture

Follows the established engine pattern: `SynthEngine` class with `subscribe`/`getSnapshot`, registers a tick callback with `TransportManager`, routes audio through `MixerEngine` channel 3. New `IR3109Processor` AudioWorklet for the filter.

## Signal Chain

```
Osc1 ──┐
Osc2 ──┤──▸ Mixer ──▸ IR3109 Filter ──▸ VCA ──▸ Mixer Ch 3
Noise ─┘        ▲              ▲           ▲
                │              │           │
               LFO ──────────(routes to pitch, cutoff, or pulseWidth)
                          Filter ADSR   Amp ADSR
```

### Oscillators

Two persistent `OscillatorNode`s running during playback. Each has independent waveform, octave offset (-2 to +2 → frequency multiplied by 0.25 to 4), and fine tune (±1 semitone). Pulse waveform implemented via `PeriodicWave` with adjustable duty cycle.

On each note, both oscillators set frequency based on `midiToFreq(note) * octaveMultiplier * tuneOffset`. Slide glides over 60ms via `exponentialRampToValueAtTime`.

Each oscillator has its own gain node for level control. Both feed into a shared summing node before the filter.

### Noise Generator

A looping white noise buffer source connected through a gain node (noise level) to the same summing node as the oscillators.

### LFO

A persistent low-frequency `OscillatorNode` (triangle or square). Rate maps 0-1 to 0.1–20Hz logarithmically. Connected via a gain node (depth) to its destination:

- `pitch` → modulates both oscillators' `detune` AudioParam
- `cutoff` → modulates the filter's `frequency` AudioParam
- `pulseWidth` → sends messages to the worklet (or modulates a parameter if pulse is implemented via PeriodicWave switching)

### IR3109 Filter (AudioWorklet)

A 4-pole OTA ladder filter. Single file: `ir3109Processor.ts` containing both the DSP class and the `AudioWorkletProcessor` wrapper. Registered as `'ir3109'` during `TransportManager.init()`.

**DSP differences from diode ladder:**
- Softer saturation: `softClip(x) = x / (1 + |x|)` instead of `tanh(x)`
- Bass compensation: `input * (1 + resonance * 0.15) - resonance * stage[3]`
- Cleaner, narrower resonance peak

**AudioParams:** `frequency` (20-20000Hz, a-rate) and `resonance` (0-4, a-rate). Same interface as diode ladder.

**Testing:** Mock `registerProcessor`, `AudioWorkletProcessor`, and `sampleRate` globals in the test file, then import and test the DSP class directly from the processor file. No separate DSP file needed.

### ADSR Envelopes

Implemented via Web Audio parameter scheduling on each note trigger:

**Note on (gate = 'note'):**
1. Cancel any scheduled values
2. Attack: `linearRampToValueAtTime(peakLevel, time + attackTime)`
3. Decay: `setTargetAtTime(sustainLevel, time + attackTime, decayTimeConstant)`

**Tie:** No re-trigger, sustain holds.

**Note off / Rest:**
1. Release: `setTargetAtTime(0, time, releaseTimeConstant)`

**Parameter mapping (0-1 to real values):**
- Attack: 0.001s – 2s (logarithmic)
- Decay: 0.001s – 2s (logarithmic)
- Sustain: 0 – 1 (linear, this is a level not a time)
- Release: 0.001s – 2s (logarithmic)

Filter ADSR modulates filter cutoff: `baseCutoff + filterEnvDepth * envelopeValue * 10000Hz`. Accent boosts the envelope depth and VCA level.

## SynthEngine Methods

**Pattern editing** (same as BassEngine):
- `setNote(step, note)`
- `toggleAccent(step)`
- `toggleSlide(step)`
- `setGate(step, gate)`

**Param editing:**
- `setOscParam(osc: 1 | 2, param: keyof OscParams, value)` — update osc1 or osc2 param
- `setFilterParam(param, value)` — cutoff, resonance, filterEnvDepth
- `setFilterEnv(param: keyof ADSRParams, value)` — filter ADSR
- `setAmpEnv(param: keyof ADSRParams, value)` — amp ADSR
- `setLFOParam(param, value)` — rate, depth, waveform, destination
- `setNoiseLevel(value)`
- `setVolume(value)`
- `setAccent(value)`

All param edits dirty `activeSoundId`. All pattern edits dirty `activePatternId`.

**Presets:**
- `loadPatternPreset(id)` / `savePatternPreset(name)` / `deletePatternPreset(id)`
- `loadSoundPreset(id)` / `saveSoundPreset(name)` / `deleteSoundPreset(id)`

## UI

### SynthSection Panel

Between the TB-303 and the Mixer.

**Layout:**

1. **SynthHeader** — "SH-2" title + "SYNTHESIZER" subtitle, pattern preset selector, sound preset selector
2. **SynthOscSection** — two side-by-side panels:
   - Each: waveform toggle (SAW/SQR/PLS), octave selector (-2 to +2), tune knob, pulse width knob, level knob
   - Between them: noise level knob
3. **SynthFilterSection** — cutoff knob, resonance knob, env depth knob, then filter ADSR (A/D/S/R knobs)
4. **SynthAmpSection** — amp ADSR (A/D/S/R knobs), volume knob
5. **SynthLFOSection** — waveform toggle (TRI/SQR), rate knob, depth knob, destination selector (PITCH/CUTOFF/PW)
6. **SynthStepGrid** — 16 steps, same as bass
7. **SynthStepEditor** — same controls as bass

### Keyboard Integration

Extend focus cycling: Tab cycles drum → bass → synth. When synth panel focused:
- `←`/`→` — step navigation
- `↑`/`↓` — pitch up/down semitone
- `S` — toggle slide
- `A` — toggle accent
- `N` — set gate to note
- `R` — set gate to rest
- `T` — set gate to tie

### Mixer

Assigns channel 3, labeled "SH-2".
