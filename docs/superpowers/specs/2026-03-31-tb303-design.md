# TB-303 Bass Synthesizer — Design Spec

## Overview

Add a TB-303-style monophonic bass synthesizer to the existing TR-909 drum machine. The 303 has its own 16-step sequencer with per-step pitch, accent, slide, and gate programming. Its signature "acid" sound comes from a sawtooth/square oscillator through a resonant diode ladder lowpass filter driven by a snappy envelope.

This feature also extracts shared transport/clock/master functionality into a standalone `TransportManager`, enabling clean multi-instrument architecture.

## Architecture

### TransportManager

A standalone class extracted from the current `AudioEngine`. It owns everything shared between instruments:

- `AudioContext` lifecycle
- Clock (Web Worker-based scheduler)
- Transport state: playing, BPM, shuffle, currentStep
- Master chain: compressor → masterGain → destination
- Master params: volume, compressor on/off, threshold, ratio, knee, attack, release

```ts
interface TransportSnapshot {
  playing: boolean;
  bpm: number;
  shuffle: number;
  currentStep: number;
  master: {
    volume: number;
    compressor: boolean;
    threshold: number;
    ratio: number;
    knee: number;
    attack: number;
    release: number;
  };
}
```

**Subscription API:** Same `subscribe`/`getSnapshot` pattern for `useSyncExternalStore`.

**Tick registration:** Instrument engines register callbacks via:
```ts
registerTickCallback(cb: (ctx: AudioContext, dest: AudioNode, time: number, step: number) => void): () => void
```

The transport calls all registered callbacks on each clock tick, passing the AudioContext, output node (compressor input or masterGain), scheduled time, and step number.

**Methods:** `init()`, `play()`, `stop()`, `setBpm()`, `setShuffle()`, `setMasterVolume()`, `setCompressorEnabled()`, `setCompressorParam()`

### DrumEngine (renamed from AudioEngine)

Owns only drum concerns. Receives `TransportManager` in constructor and registers a tick callback.

```ts
interface DrumSnapshot {
  pattern: {
    steps: Record<InstrumentId, boolean[]>;
    accents: boolean[];
  };
  instruments: Record<InstrumentId, InstrumentParams>;
  presets: {
    patterns: PatternPreset[];
    kits: KitPreset[];
    activePatternId: string | null;
    activeKitId: string | null;
  };
}
```

No transport or master state — those live in TransportManager. All existing drum functionality (pattern editing, instrument params, presets, hi-hat choke) works the same, just routed through the transport's output node.

### BassEngine (new)

Owns all 303 concerns. Receives `TransportManager` in constructor and registers a tick callback.

```ts
interface BassSnapshot {
  pattern: BassPattern;
  synth: SynthParams;
  presets: {
    patterns: BassPatternPreset[];
    synths: BassSynthPreset[];
    activePatternId: string | null;
    activeSynthId: string | null;
  };
}
```

## 303 Sequencer Data Model

```ts
interface BassStep {
  note: number;       // MIDI note number (0-127), e.g. 36 = C2
  accent: boolean;
  slide: boolean;     // portamento to next note
  gate: 'note' | 'rest' | 'tie';  // tie extends previous note
}

interface BassPattern {
  steps: BassStep[];  // 16 steps
}
```

**Gate behavior:**
- `note` — trigger the envelope, play the note
- `rest` — silence, no envelope trigger
- `tie` — extend the previous note without re-triggering the envelope
- `slide` — when true and the next step is a note or tie, glide the oscillator frequency to the next pitch over ~60ms

## 303 Synthesis

### Signal Chain

Oscillator → DiodeLadderWorklet → VCA → transport output node

### Oscillator

Persistent `OscillatorNode` that runs continuously during playback. Waveform is switchable between sawtooth and square. Note changes set the frequency directly (or glide via `exponentialRampToValueAtTime` for slides).

### SynthParams

```ts
interface SynthParams {
  waveform: 'sawtooth' | 'square';
  cutoff: number;       // 0–1, maps to filter frequency (20–20000 Hz, logarithmic)
  resonance: number;    // 0–1, maps to filter feedback (0–4)
  envMod: number;       // 0–1, envelope modulation depth on filter cutoff
  decay: number;        // 0–1, envelope decay time (30ms–2s)
  accent: number;       // 0–1, accent boost amount for volume + filter
  volume: number;       // 0–1, output level
}
```

### Filter Envelope

On each new note (not rest, not tie):
1. Cutoff jumps to `baseCutoff + envMod * envelopeAmount` (boosted further by accent)
2. Exponentially decays back to `baseCutoff` over `decay` time

On accent steps, both the filter envelope depth and VCA level are boosted by the `accent` parameter.

On slide steps, the oscillator frequency ramps smoothly to the next note's pitch over ~60ms rather than jumping instantly.

### Diode Ladder Filter (AudioWorklet)

A custom `AudioWorkletProcessor` implementing a 4-pole cascaded one-pole filter with nonlinear feedback, modeling the diode ladder topology of the 303's filter.

**DSP algorithm:** 4 cascaded stages, each:
```
y[n] = y[n-1] + g * (tanh(x[n]) - tanh(y[n-1]))
```

Where `g` is derived from the cutoff frequency using the bilinear transform pre-warp. The feedback path takes the 4th stage output, scales by resonance, and subtracts from the input.

**AudioParams:**
- `frequency` — cutoff in Hz (20–20000), a-rate automatable for envelope
- `resonance` — feedback amount (0–4), a-rate automatable

**Files:**
- `src/engine/bass/diodeLadderDSP.ts` — pure DSP math functions, testable without AudioWorklet
- `src/engine/bass/diodeLadderProcessor.ts` — AudioWorkletProcessor wrapper
- Registered during `TransportManager.init()` via `audioContext.audioWorklet.addModule()`

## BassEngine Methods

**Pattern editing:**
- `setNote(step: number, note: number)` — set MIDI note for step
- `toggleAccent(step: number)` — toggle accent on/off
- `toggleSlide(step: number)` — toggle slide on/off
- `setGate(step: number, gate: 'note' | 'rest' | 'tie')` — set gate type

**Synth params:**
- `setSynthParam(param: keyof SynthParams, value: number | string)` — update any synth parameter
- `setWaveform(wf: 'sawtooth' | 'square')` — toggle oscillator waveform

**Presets:**
- `loadPatternPreset(id)` / `savePatternPreset(name)` / `deletePatternPreset(id)`
- `loadSynthPreset(id)` / `saveSynthPreset(name)` / `deleteSynthPreset(id)`

**Dirty tracking:** Same approach as DrumEngine — editing pattern nullifies activePatternId, editing synth params nullifies activeSynthId.

### Built-in Pattern Presets (6)

| Name | Character |
|------|-----------|
| Acid Line | Ascending C minor pentatonic with slides and accents |
| Resonance Workout | Wide intervals, heavy slides and accents — filter showcase |
| Bubbling | Fast ties and rests creating a rhythmic bubbling pattern |
| Sub Bass | Low octave, quarter notes, minimal slides — deep foundation |
| Staccato Funk | Short gated notes, syncopated, no slides |
| Arpeggiated | Ascending/descending triad with slides on peaks |

### Built-in Synth Presets (6)

| Name | Character |
|------|-----------|
| Classic Acid | Sawtooth, medium cutoff, high resonance, high env mod |
| Squelch | Square, low cutoff, max resonance, max env mod |
| Warm Bass | Sawtooth, low cutoff, low resonance, low env mod |
| Screamer | Sawtooth, high cutoff, high resonance, fast decay |
| Hollow | Square, medium cutoff, medium resonance |
| Dark Sub | Sawtooth, very low cutoff, no resonance, no env mod |

## UI

### BassSection Panel

Below the master section, visually distinct but same design language.

**Layout (top to bottom):**

1. **BassHeader** — "TB-303" title, pattern preset selector, synth preset selector, SAW/SQR waveform toggle
2. **BassKnobs** — Cutoff, Resonance, Env Mod, Decay, Accent, Volume (same knob component style as MasterSection)
3. **BassStepGrid** — 16 steps grouped in 4s. Each step shows:
   - Note name (e.g. "C2", "F#3") — drag up/down to change pitch
   - Accent indicator (orange dot)
   - Slide indicator ("S" or curve)
   - Gate state: filled = note, empty = rest, connected/striped = tie
4. **BassStepEditor** — controls for the selected step: accent toggle, slide toggle, gate type buttons (Note/Rest/Tie), octave up/down

### Component Tree

```
BassSection
├── BassHeader          — title, preset selectors, waveform toggle
├── BassKnobs           — synth parameter knobs
├── BassStepGrid        — 16-step sequencer display
└── BassStepEditor      — selected step editing controls
```

### Keyboard Integration

Extend the single `document.addEventListener('keydown', ...)`:

- `Tab` — switch focus between 909 and 303 panels
- When 303 panel is focused:
  - `←` / `→` — step navigation
  - `↑` / `↓` — pitch up/down (semitone)
  - `S` — toggle slide
  - `A` — toggle accent
  - `N` — set gate to note
  - `R` — set gate to rest
  - `T` — set gate to tie
